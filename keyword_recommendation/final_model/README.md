## final_model 최종 모델

### 📌 1. 개요
초기 모델은 텍스트 기반 `TF-IDF`를 활용하여 키워드를 추출하고,<br>
`Word2Vec`을 통해 연관 키워드를 추천하는 방식<br>
하지만, 사용자 반응(조회수)이 반영되지 않는 한계가 존재<br>
최종 모델에서는 이를 개선하여 최종 키워드 추천 모델을 구축
<br>
<br>

### 📁 2. 데이터 파일 설명
- `baseline05.ipynb`
  - 최종 모델 파이썬 코드 파일
- `total_blog(12_3_49).csv`
  - 블로그 게시물 수동 크롤링 데이터
  - 게시물 게시 일주일간의 조회수 합계
  - 의미 : 12/3에 수집한 49개의 게시물
<br>

### ⚙️ 3. 주요 개선점
**1. 고성과 데이터 기반 필터링**
- 전체 게시글이 아닌 조회수 상위 50% 게시글만 선별
- 단순 빈도가 아닌 성과 기반 데이터 중심 분석 구조로 개선

**2. 텍스트 전처리 고도화**
- 불용어 제거 및 2글자 이상 키워드만 필터링

**3. LDA 기반 토픽 모델링 적용**
- TF-IDF 기반 LDA 모델 학습
- Coherence Score 기반 최적 토픽 수(k) 탐색
- 토픽 간 유사도 계산 -> 중복 토픽 제거

**4. 핵심 키워드 + 연관 키워드 확장**
- 토픽별 핵심 키워드 추출
- Word2Vec을 활용하여 의미 기반 연관 키워드 확장

**5. 토픽 중요도 기반 키워드 가중화**
- 토픽별 문서 수 + Coherence Score 정규화
- 두 값을 결합하여 토픽 중요도 점수 산출
- 중요도에 따라 추천 키워드 개수 동적 조정

**6. 외부 데이터 기반 트렌드 반영**
- 네이버 데이터랩 API 활용
- 키워드 검색량 기반 토픽별 수요 점수 계산
- Min-Max / 합계 기반 정규화를 통해 토픽 간 상대적 수요 비교 가능하도록 개선
<br>

### 📊 4. 결과
- 토픽 기반 핵심 키워드 도출
- Word2Vec 기반 연관키워드 확장
- 외부 검색량 기반 수요 점수 산출
- 키워드 조합을 통한 콘텐츠 주제 후보 도출 프로세스 구축
<br>

### 🚀 5. 초기모델 대비 개선점
- TF-IDF 단일 키워드 → **토픽 중심 구조로 확장**
- 단순 빈도 기반 → **성과 데이터 기반**
- 정적 키워드 → **의미 기반 키워드 확장 (Word2Vec)**
- 내부 데이터 → **외부 트렌드 반영**
<br>

### 💡 6. 활용 가능성
- 신규 게시물 데이터 추가 시 **반복 적용 가능**한 구조
- 데이터 기반 콘텐츠 기획 자동화 확장 가능
